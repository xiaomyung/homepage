~/football/renderer diagnostic

One full-width strip per concept. Each strip has freecam enabled via the renderer's debug cam — drag to orbit, WASD to pan on the ground plane, Q/E to raise/lower, wheel to zoom, R to reset to the default close-up pose. Every animated strip loops its own state and redraws each frame.

full scene (live tick)

Default showcase camera. Physics steps every frame, both players + ball animate.

walk cycle (phase cycling)

Both players parked with walk amplitude = 1.0 and phase cycling through a full TWO_PI so arm + leg swing is visible.

stamina fill (cycling 0 → 1)

Player stamina sweeps 0 → 1 → 0 repeatedly. Torso body capsule fills from hip to neck; above the fill line only the Fresnel-rim silhouette survives. Limbs, head, and fists stay monochrome solid throughout.

heading rotation (angle cycling)

Heading rotates continuously through a full circle. Shoulders / hips / head / limbs stay in lock-step under the rotating local frame.

push (looping state machine)

pushTimer is re-armed whenever it clears so the scripted arm / pivot / body-dip / hop curve runs continuously.

kick (ground, looping)

Kick state machine re-armed each time it clears — windup → strike → recovery on loop.

airkick (leap, looping)

Airkick forced on with player.airZ driven by the same sin-arc the physics uses.

celebration

pauseState='celebrate' + goalScorer set so the jumping-jack pose runs continuously.

airborne ball + bounce particles

Ball held aloft with a synthetic bounce event injected every few ticks so the particle system fires continuously.

goal burst

Injected 'goal' event spawns the full particle burst out of the left goal mouth. Re-injected every ~120 frames.
