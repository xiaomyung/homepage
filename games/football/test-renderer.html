~/football/renderer diagnostic

Visual smoke tests for renderer.js. Each strip instantiates an independent Renderer pointing at a shared physics state it controls directly — nothing talks to the broker. Use this page to verify stickman animation poses, particles, shadows, and camera behaviour after renderer changes.

full scene (live tick)

Default showcase camera. Physics steps every frame, both players + ball animate.

stickman poses (amplitude = 1.0, varying phase)

Each frame pins a different walk-cycle phase so arms and legs are held at a specific swing angle.

push (scripted curve)

pushTimer is forced non-zero so the scripted arm / pivot / body-dip / hop curve drives the pose.

celebration

pauseState='celebrate' + goalScorer set to each player in turn.

airborne ball + bounce particles

Ball held aloft with a synthetic bounce event injected each tick so the particle system fires continuously.

goal burst

Injected 'goal' event spawns the full ~42-particle burst out of the left goal mouth.
